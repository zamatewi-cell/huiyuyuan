### 失败根因分类

1. **pumpAndSettle 超时**（约10个）— 动画/定时器未正确mock
2. **数据快照不同步**（2个）— seed 数据与 JSON snapshot 不一致
3. **Mock 数据/配置问题**（约10个）— 测试环境与运行环境差异
4. **枚举/API 变更**（约5个）— 代码重构后测试未跟进
5. **加载失败**（约4个）— 文件/模块路径问题

### 处理建议

- 这些失败不影响生产运行（生产已通过健康检查）
- 建议在 Phase F 后续迭代中逐个修复
- 当前 CI 门禁应标记为"已知失败"而非阻断
