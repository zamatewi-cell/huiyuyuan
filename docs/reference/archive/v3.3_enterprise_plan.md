## 二、v3.3 执行计划

### Phase 1: 后端订单全生命周期 (Critical Path)

> 目标: 补全7个缺失端点 + 库存扣减 + 支付模拟网关

#### 1.1 库存原子扣减
```
POST /api/orders → 下单时 stock -= quantity，失败则 400
```

#### 1.2 支付网关模拟
```
POST   /api/orders/{id}/pay        → 创建支付记录，3秒后自动回调成功
GET    /api/orders/{id}/pay-status  → 轮询支付状态 (pending/success/failed)
POST   /api/pay/callback            → 支付回调(内部调用)
```

#### 1.3 订单状态变更
```
POST   /api/orders/{id}/cancel          → 取消订单 + 恢复库存
POST   /api/admin/orders/{id}/ship      → 管理员发货(carrier+trackingNo)
POST   /api/orders/{id}/confirm-receipt → 确认收货
POST   /api/orders/{id}/refund          → 申请退款(reason)
```

#### 1.4 查询端点
```
GET    /api/orders/{id}/logistics   → 返回物流轨迹列表
GET    /api/orders/stats            → 订单统计(按用户)
GET    /api/admin/dashboard         → 管理端统计面板
GET    /api/admin/activities        → 最近操作动态(从真实订单生成)
```

### Phase 2: 支付页面(PaymentScreen) 接入 PaymentService

> 目标: 创建独立支付页面，完整接入已有的 PaymentService

#### 2.1 创建 `screens/payment/payment_screen.dart`
- 显示订单信息卡片(订单号/商品/金额)
- 支付方式选择(微信/支付宝/余额，调用 PaymentService)
- 15分钟倒计时
- "确认支付"按钮 → PaymentService.createPaymentOrder() → 轮询状态
- 支付成功动画 → 跳转订单详情
- 支付失败重试

#### 2.2 重构 CheckoutScreen 支付流程
- `_processPayment()` 不再直接 simulatePayment
- 改为: createOrder → Navigator.push(PaymentScreen) → PaymentScreen 处理支付
- PaymentScreen 成功后才更新订单状态为 paid

#### 2.3 订单列表"立即付款"接入
- `_showPaymentDialog` → 改为跳转 PaymentScreen 而非 AlertDialog

### Phase 3: 加购流程修复 + 数据打通

#### 3.1 修复 product_detail_screen._addToCart()
- 从 `StorageService.addToCart(product.toJson())` 改为 `ref.read(cartProvider.notifier).addItem(CartItemModel(...))`
- 保证 UI 状态与存储同步

#### 3.2 收藏列表接入真实API
- `favoritesProvider` 改为调用 `/api/favorites` 获取完整商品数据
- 移除 `_getMockProduct()` 方法

#### 3.3 Profile 资产卡片动态化
- 从 `orderStatsProvider` 获取"今日订单"和"待处理"
- 本月业绩从订单金额汇总

#### 3.4 OperatorHome 数据动态化
- 统计项从 `orderStatsProvider` 计算
- 待办从未发货订单列表生成

#### 3.5 仪表盘活动流动态化
- `_getMockActivities()` → 从最近订单状态变更生成真实动态

### Phase 4: 店铺雷达 + 图片选择器

#### 4.1 店铺详情页实现
- 创建 `screens/shop/shop_detail_screen.dart`
- 店铺信息/评分/商品列表/地图位置

#### 4.2 图片选择器完善
- `_pickFromGallery()` / `_pickFromCamera()` 接入 image_picker
- 图片预览+裁剪

### Phase 5: 创新功能

#### 5.1 ? 订单状态实时追踪页(OrderTrackingScreen)
- 可视化状态流程图: 下单→支付→发货→运输→签收
- 每个节点带时间戳+动画
- 支持分享物流截图

#### 5.2 ? 智能补货建议卡片
- 基于库存安全线和销售趋势
- 在管理员仪表盘显示"建议补货"卡片

#### 5.3 ? 商品价格走势图

#### 5.4 ? 客户分层标签系统
