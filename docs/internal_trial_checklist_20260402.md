# 汇玉源内部试运行 Checklist

> 版本：2026-04-02
> 适用范围：3-10 人规模内部试运行

---

## 1. 上线前必做

- [ ] 所有内部成员首次访问前强刷浏览器缓存
- [ ] 管理员账号、操作员账号、普通用户账号至少各准备 1 个
- [ ] 管理员先登录一次，确认后台首页、商品、库存、订单可正常打开
- [ ] 至少抽查 10 个商品卡片和 5 个商品详情页，确认没有明显中英混排
- [ ] 确认地址管理、支付账户、订单列表、物流页可以正常打开
- [ ] AI 助手能正常进入，在线状态和欢迎语正常

---

## 2. 支付前必做

- [ ] 明确当前试运行的收款主体是谁
- [ ] 如果是平台管理员统一收款，先用管理员账号进入“收款账户”页，至少配置 1 个默认收款账户
- [ ] 默认收款账户至少补齐一种真实可用方式：
  - [ ] 微信收款码
  - [ ] 支付宝收款码
  - [ ] 银行卡信息
- [ ] 如果配置了二维码，确认支付页能正常加载二维码图片
- [ ] 管理员从后台首页的“订单管理”入口进入一次，确认能打开订单列表
- [ ] 用普通用户下 1 笔小额订单，进入支付页，确认能看到收款账户信息
- [ ] 普通用户完成转账后，管理员进入订单管理，对该订单执行“确认到账”
- [ ] 订单状态确认能从 `pending` 变成 `paid`
- [ ] 管理员再继续执行发货，用户侧可查看物流并确认收货
- [ ] 至少准备 1 个可用收款方式：
  - [ ] 微信收款码
  - [ ] 支付宝收款码
  - [ ] 银行卡信息
- [ ] 收款二维码链接或图片可稳定访问

---

## 3. 每日试运行检查

- [ ] 登录页语言显示正常
- [ ] 商品列表首屏图片正常
- [ ] 商品详情页 `Product Info / Description / Reviews` 正常
- [ ] 新下订单可进入订单列表
- [ ] 买家支付页能看到正确的收款账户 / 收款码
- [ ] 管理员确认到账后订单状态可从 `pending` 变成 `paid`
- [ ] 管理员端可以发货
- [ ] 用户端可以看到物流
- [ ] 用户端可以确认收货
- [ ] 评价页可以打开并提交

---

## 4. 重点人工巡检项

- [ ] 登录页有没有再出现 `???` 或翻译 key
- [ ] 商品标题有没有再次出现中英混排
- [ ] 商品详情页有没有出现中文残留
- [ ] 地址页、支付页、隐私政策、用户协议语言是否正常
- [ ] AI 页首句、输入框、在线/离线状态是否正常
- [ ] 管理端库存页、商品页、订单页是否正常切语言

---

## 5. 当前已知限制

- 当前支付链路已经支持“订单绑定收款账户 + 支付页展示收款码 + 管理员确认到账”
- 但它仍然不是第三方支付自动回调入账，依然属于内部人工确认支付模式
- 因此，内部试运行时仍需要明确谁负责看款、谁负责确认到账、谁负责发货

---

## 6. 试运行建议

- 先跑 3-5 天内部真实流程
- 每天记录：
  - 新发现的漏翻页面
  - 新商品命名翻译异常
  - 支付流程卡点
  - 库存/订单状态异常
- 试运行期间不要频繁改数据库结构，优先收敛真实使用反馈
